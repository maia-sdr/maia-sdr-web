Env varCurrent valueStatusRequiredAction
"Unset maia-sdr overrides" clears ramboot_verbose, qspiboot_verbose, and qspiboot so the bootloader defaults apply again. You do not need to clear these before reverting to stock ADI firmware — the uio_pdrv_genirq kernel argument is a no-op there. attr_name, attr_val, and mode are left in place because they describe your hardware regardless of firmware.

Networking (u-boot environment)

VariableValueAction
Leave a field blank and click Apply to unset the variable (falls back to the bootloader default). A reboot is required for any change to take effect.

USB ethernet compatibility mode

Pick the row matching your host OS. Writes the u-boot usb_ethernet_mode variable (rndis / ncm / ecm). A reboot is required to apply the change. RNDIS is the firmware default; unset also means RNDIS.

Raw /opt/config.txt editor

Escape hatch for settings that aren't exposed as their own control (hostname, per-interface IPs, xo_correction, etc.). USB ethernet mode is better set from the panel above — it uses the u-boot env and doesn't need a file write. Saves go via chunked base64 upload so any content is safe. A reboot is required for changes to take effect.
